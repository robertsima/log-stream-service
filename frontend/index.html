      How it works

      PrairieLog API sends you important alerts, here's the flow:

      1. Alert webhook

        Your Slack or Discord webhook URL — where you want PrairieLog to send alerts.

      2. Ingestion token

        A secret your app uses to send logs to PrairieLog. You will want to hang onto it once you generate one.

      3. Send ERROR logs

        Install the TypeScript, Python, or Logback SDK, or POST JSON logs to the batch ingestion endpoint with the X-Ingestion-Token header.

      4. Get alerts

        Matching ERROR logs are aggregated, then delivered to your alert webhook. Other levels are accepted but stay quiet.

      Add to your app

      Your app or service sends ERROR logs to PrairieLog; PrairieLog forwards aggregated alerts/messages to the Slack or Discord webhook you configure.

      1. Paste your alert webhook in the live demo to verify Slack/Discord delivery
      2. Copy your ingestion token and SDK starter code from the demo
      3. Call it from your app or backend when an ERROR occurs
      Node / server-side 
      import { PrairieLogClient } from "@prairielog/client";

const prairieLog = new PrairieLogClient({
  apiUrl: "__API_BASE_URL__",
  ingestionToken: process.env.PRAIRIELOG_INGESTION_TOKEN,
  defaultLogger: "my-app",
  batchSize: 50
});

prairieLog.installNodeHandlers();

      About PrairieLog API

      PrairieLog is a lightweight relay. Logs are held in memory briefly for aggregation, then alerts go out to your Slack or Discord channel. Logs are not stored long-term.

      Ingestion tokens are shown once at creation. Save yours in a server environment variable — not in browser code or git.

      Production apps should send logs from the backend only, this portal is not a client side logging service.

      Frequently asked questions

      What is PrairieLog?

      PrairieLog is a lightweight log alerting service for small apps. Your application sends log events to a webhook-style ingestion endpoint (directly or through an SDK), and PrairieLog delivers aggregated real-time error alerts to Slack or Discord. There is no log warehouse to run and nothing to self-host.

      How do Slack and Discord error alerts work?

      You register a Slack or Discord incoming webhook as an alert destination for your app. When matching ERROR logs arrive, PrairieLog groups them over a short aggregation window (about one minute) and sends one summarized alert to your channel instead of a message per log line.

      Does PrairieLog store my logs?

      No. Logs are held in memory briefly for aggregation and then discarded — PrairieLog is a relay, not a log warehouse. If you need long-term retention or search, pair it with a storage-focused tool.

      Which languages and loggers are supported?

      There are SDKs for TypeScript/JavaScript (browser, Node, React, Angular), a Python logging handler, and a Java Logback appender. The ingestion API also accepts tolerant JSON from common logging stacks like Logstash, python-json-logger, Winston, and Pino without client-side mapping. The SDKs are currently installed from the repository — registry packages (npm, PyPI, Maven Central) are not published yet.

      Who is PrairieLog for?

      Indie developers and small teams who want simple log monitoring and real-time error alerts without running observability infrastructure. If a full APM or log warehouse is overkill for your app, PrairieLog covers the "tell me in Slack when something breaks" part. It is currently free to use.
