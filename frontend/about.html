About PrairieLog API

      PrairieLog is a lightweight relay for small applications that need to know when something breaks, without adopting a full observability stack.

      Start with an SDK Try the Demo

      What it does

      PrairieLog accepts log events from your app, groups matching ERROR logs over a short window, and posts a summarized alert to the Slack or Discord webhook you configure.

      Logs are held in memory briefly for aggregation, then alerts go out to your channel. PrairieLog is intentionally not a long-term log store.

      Design principles

      Alert-first

      Focus on the signal you actually need in chat: grouped ERROR alerts with useful context.

      SDK-first

      Use TypeScript, Python, or Java Logback SDKs for batching, level normalization, and required fields.

      Small by design

      No Kafka, Redis, Elasticsearch, or warehouse requirement for the core relay path.

      Secrets and data handling

      • Ingestion tokens are shown once at creation. Save yours in a server environment variable, not in browser code or git.
      • Full alert webhook URLs should be treated as secrets and are not meant to be displayed after creation.
      • Production apps should send logs from backend or server-controlled contexts. Browser tokens are visible to users.
      • PrairieLog relays alerts; it does not provide long-term retention, search, or compliance archiving.

      Who it is for

      PrairieLog is for indie developers and small teams who want simple log monitoring and real-time error alerts without running observability infrastructure. If a full APM platform is overkill, PrairieLog covers the "tell me in Slack or Discord when something breaks" workflow.
